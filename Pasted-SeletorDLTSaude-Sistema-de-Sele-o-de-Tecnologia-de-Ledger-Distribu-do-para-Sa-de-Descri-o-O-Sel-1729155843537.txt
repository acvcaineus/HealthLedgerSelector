SeletorDLTSaude - Sistema de Seleção de Tecnologia de Ledger Distribuído para Saúde
Descrição: O SeletorDLTSaude é uma aplicação interativa desenvolvida para ajudar profissionais e pesquisadores a escolherem a melhor solução de Distributed Ledger Technology (DLT) e o algoritmo de consenso mais adequado para projetos de saúde. A aplicação guia o usuário por meio de um fluxo de perguntas, considerando o cenário de implementação (como Registros Médicos Eletrônicos, Cadeia de Suprimentos ou Consentimento de Pacientes) e as características específicas do projeto.

A aplicação utiliza o framework Streamlit para uma interface amigável e visualizações dinâmicas com Plotly. As recomendações são personalizadas com base nas respostas fornecidas e são salvas em um banco de dados SQLite, garantindo que os usuários possam acessar suas recomendações anteriores.

Principais Funcionalidades:
Escolha do Cenário de Implementação: O usuário pode escolher entre diferentes cenários de aplicação para DLT na saúde, como Registros Médicos Eletrônicos (EMR), Cadeia de Suprimentos Farmacêutica ou Consentimento de Pacientes.
Perguntas Guiadas: A aplicação faz uma série de perguntas sequenciais com base no cenário escolhido, ajudando o usuário a definir os requisitos necessários para selecionar a DLT e o algoritmo de consenso mais adequado.
Recomendações Personalizadas: Com base nas respostas, a aplicação oferece uma recomendação personalizada de framework DLT e algoritmo de consenso, com explicações claras sobre o motivo da escolha.
Salvamento de Recomendações: O sistema utiliza um banco de dados SQLite para armazenar as recomendações feitas para cada usuário, permitindo que ele acesse suas recomendações anteriores e visualize seu histórico de escolhas.
Visualização de Respostas: Ao final do processo, o sistema exibe um gráfico de barras com as respostas fornecidas, ajudando o usuário a visualizar como suas respostas influenciaram a recomendação.
Login e Registro de Usuários: A aplicação permite que os usuários se registrem e façam login para salvar e acessar suas recomendações personalizadas.
Como Usar o SeletorDLTSaude:
Registro ou Login: Crie sua conta ou faça login para começar a usar o SeletorDLTSaude.
Escolha o Cenário de Implementação: Escolha o cenário de saúde para o qual você deseja implementar uma DLT, como EMR, Cadeia de Suprimentos, etc.
Responda às Perguntas: O sistema guiará você por uma série de perguntas sobre seus requisitos específicos para a implementação da DLT.
Obtenha a Recomendação: Após responder às perguntas, o sistema recomendará a melhor DLT e o algoritmo de consenso, com base nas suas respostas.
Salve e Acesse Suas Recomendações: Salve suas recomendações para acessá-las novamente no futuro, através do seu perfil.
Visualize Gráficos das Respostas: Visualize as respostas fornecidas em um gráfico interativo.

Dependências:
Streamlit: Framework para construir a interface do usuário de maneira simples e rápida.
SQLite: Banco de dados leve e incorporado para armazenamento de dados de usuários e recomendações.
Plotly: Biblioteca de visualização de dados para gerar gráficos dinâmicos das respostas dos usuários.
Pandas: Biblioteca para manipulação de dados e processamento das respostas do usuário.
Estrutura de Arquivos do Projeto:
main.py: Contém o código principal da interface com o Streamlit, que guia o usuário pelo fluxo de perguntas e exibe as recomendações.
decision_logic.py: Contém a lógica de decisão com base no fluxo de perguntas para determinar a DLT e o algoritmo de consenso.
dlt_data.py: Contém as perguntas e dados sobre DLTs e algoritmos de consenso.
user_management.py: Contém as funcionalidades para registro, login e gerenciamento de sessões dos usuários.
database.py: Gerencia a criação e manipulação das tabelas de usuários e recomendações no banco de dados SQLite.
requirements.txt: Lista de dependências necessárias para rodar o projeto.
Exemplo de Uso:
Escolha do Cenário: O usuário escolhe o cenário de aplicação, como Cadeia de Suprimentos.
Perguntas Guiadas: O usuário responde a perguntas como "A privacidade é crítica?" ou "A rede precisa de alta escalabilidade?".
Recomendação: Com base nas respostas, o sistema pode recomendar um Blockchain Permissionado com o algoritmo de consenso Proof of Stake para garantir privacidade e eficiência.
Visualização: O usuário vê um gráfico das respostas fornecidas e pode salvar a recomendação no banco de dados.
